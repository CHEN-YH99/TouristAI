# 贡献指南

感谢你对TouristAI项目的关注！我们欢迎任何形式的贡献。

## 🤝 如何贡献

### 报告Bug
如果你发现了Bug，请：
1. 检查是否已有相关Issue
2. 创建新Issue，包含：
   - Bug描述
   - 复现步骤
   - 预期行为
   - 实际行为
   - 环境信息

### 提出功能建议
如果你有好的想法，请：
1. 创建Feature Request Issue
2. 描述功能需求
3. 说明使用场景
4. 提供设计建议（可选）

### 提交代码
如果你想贡献代码，请：
1. Fork项目
2. 创建功能分支
3. 编写代码
4. 提交Pull Request

## 📝 开发流程

### 1. Fork项目
点击右上角的Fork按钮

### 2. 克隆到本地
```bash
git clone https://github.com/YOUR_USERNAME/TouristAI.git
cd TouristAI
```

### 3. 创建分支
```bash
git checkout -b feature/your-feature-name
```

### 4. 开发
按照[开发指南](DEVELOPMENT.md)配置环境并开发

### 5. 提交
```bash
git add .
git commit -m "feat: 添加新功能"
git push origin feature/your-feature-name
```

### 6. 创建Pull Request
在GitHub上创建Pull Request

## 📋 代码规范

### TypeScript/JavaScript
- 使用TypeScript
- 遵循ESLint规则
- 使用Prettier格式化
- 添加必要的注释

### Python
- 遵循PEP 8
- 使用类型注解
- 添加docstring
- 保持代码简洁

### 提交信息
使用约定式提交：
```
feat: 添加新功能
fix: 修复bug
docs: 更新文档
style: 代码格式调整
refactor: 代码重构
test: 添加测试
chore: 构建/工具相关
```

## 🧪 测试

### 运行测试
```bash
# 前端测试
cd frontend
npm run test

# 后端测试
cd backend
npm run test
```

### 添加测试
- 为新功能添加测试
- 确保测试通过
- 保持测试覆盖率

## 📚 文档

### 更新文档
如果你的改动影响到：
- 功能使用
- API接口
- 配置方式
- 部署流程

请同时更新相关文档。

### 文档规范
- 使用Markdown格式
- 保持简洁清晰
- 添加代码示例

## 🔍 代码审查

### Pull Request要求
- 清晰的描述
- 相关的Issue链接
- 测试通过
- 代码规范
- 文档更新

### 审查流程
1. 自动化测试
2. 代码审查
3. 讨论和修改
4. 合并到主分支
